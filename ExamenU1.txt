PASO 1: Creación de la Interfaz de Métodos de Pago
Todo buen diseño basado en SOLID empieza definiendo "contratos" (interfaces). Esto nos va a preparar para no depender de implementaciones concretas.
Lo que debes hacer:
1.	En tu proyecto SOLIDPrinciples, asegúrate de tener una carpeta llamada Interfaces dentro de la carpeta Application.
2.	Dentro de la carpeta Interfaces, crea un archivo llamado IPaymentMethodRAJA.cs.
3.	Pega el siguiente código en ese archivo:

namespace SOLIDPrinciples.Application.Interfaces;

public interface IPaymentMethodRAJA
{
    void ProcessPaymentRAJA(double amountRAJA);
    string GetMethodNameRAJA();
}

¿Qué principio SOLID estamos aplicando aquí y por qué?
1.	Principio de Inversión de Dependencias (DIP): Este principio dice que los módulos de alto nivel (como el gestor que procesará los pagos) no deben depender de los módulos de bajo nivel (como el pago por "Tarjeta" o "Efectivo"), sino de abstracciones. Al crear esta interfaz, nuestro sistema central dependerá de IPaymentMethodRAJA y no de las clases específicas.
2.	Principio de Abierto/Cerrado (OCP - Preparación): Al tener esta interfaz, en el futuro podremos añadir 10 formas nuevas de pago (Criptomonedas, Cheques, etc.) simplemente creando clases nuevas que implementen esta interfaz, sin tener que tocar o modificar el código que ya existe realizando los cobros.

PASO 5: Creación del Servicio de Procesamiento con Inyección de Dependencias
Este es el corazón de la aplicación. Aquí uniremos el pago y la notificación basándonos en lo que nos pide el examen: "Se debe aplicar inyección de dependencias en el constructor" y "No usar genéricos... puedes usar arreglos normales".
Dado que queremos enviar notificaciones por diferentes canales a la vez, pasaremos un arreglo normal ([]) de notificaciones, cumpliendo así la restricción de no usar List<T>.
Lo que debes hacer:
1.	En tu proyecto SOLIDPrinciples, asegúrate de tener una carpeta llamada Services dentro de la carpeta Application.
2.	Dentro de la carpeta Services, crea un archivo llamado PaymentProcessorRAJA.cs.
3.	Pega el siguiente código:


namespace SOLIDPrinciples.Application.Services;

using System;
using System.Collections.Generic;
using System.Text;
using SOLIDPrinciples.Application.Interfaces;
using SOLIDPrinciples.Application.Models;

public class PaymentProcessorRAJA
{
    private readonly IPaymentMethodRAJA _paymentMethodRAJA;
    private readonly INotificationChannelRAJA [] _notificationChannelRAJA;

    // Constructor que inyecta la estrategia de pago y los canales de notificación

    public PaymentProcessorRAJA(IPaymentMethodRAJA paymentMethodRAJA, INotificationChannelRAJA[] notificationChannelRAJA)
    {
        _paymentMethodRAJA = paymentMethodRAJA;
        _notificationChannelRAJA = notificationChannelRAJA;
    }
    public void ExecutePaymentRAJA(PaymentRequestRAJA paymentRequestRAJA)
    {
        Console.WriteLine($"\n--- Iniciando transaccion para : {paymentRequestRAJA.UserNameRAJA} ---");

        //Procesar el pago a traves de la interfaz (sin importar si es tarjeta, efectivo, etc)
        _paymentMethodRAJA.ProcessPaymentRAJA(paymentRequestRAJA.AmountRAJA);

        string resumenRAJA = $"Pago de {paymentRequestRAJA.UserNameRAJA} procesado exitosamente mediante {_paymentMethodRAJA.GetMethodNameRAJA}";
        Console.WriteLine(resumenRAJA);

        // 2. Notificamos por cada canal configurado (Usando un arreglo simple, no List)
        Console.WriteLine(" --- Enviando notificaciones ---");
        for (int i = 0; i < _notificationChannelRAJA.Length; i++)
        {
            _notificationChannelRAJA[i].SendNotificationRAJA(resumenRAJA);
        }
        Console.WriteLine("--------------------------------------------------\n");
    }
}
